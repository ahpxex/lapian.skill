# 教程 (Tutorial)

叠加在 `_base.md` 之上的题材特化分析维度。分析重心在"为什么有效"。

## 1. 认知负荷管理

- **前置知识假设**: 假设观众知道什么？假设是否合理？
- **概念引入节奏**: 新概念的引入频率？是否给足了消化时间？
- **复杂度曲线**: 难度是线性增加还是有阶梯？
- **认知锚点**: 是否用已知概念解释未知概念？

## 2. 分步节奏

- **步骤粒度**: 每一步的信息量是否适当？太大则跟不上，太小则啰嗦？
- **完成感设计**: 每一步完成后是否有可见的成果反馈？
- **错误预防**: 是否提前警告常见错误？
- **可跟做性**: 观众能否暂停视频并实际操作？节奏是否允许？

## 3. 示范与讲解的配比

- **show vs tell**: 直接演示和口头解释的比例？
- **屏幕/操作录制**: 屏幕录制的清晰度、光标高亮、放大处理？
- **同步性**: 操作和讲解是否同步？还是先讲再做/先做再讲？
- **结果预览**: 是否先展示最终结果再教步骤？

## 4. 结构设计

- **大纲透明**: 是否在开头告知整体结构和预期成果？
- **模块化**: 内容是否可以模块化消费？还是必须从头看到尾？
- **总结与回顾**: 是否有阶段性总结和最终回顾？
- **延伸资源**: 是否提供了进一步学习的路径？

## 5. 可及性

- **多水平适配**: 是否同时照顾了新手和有经验的观众？
- **语言清晰度**: 解说是否清晰、无歧义？
- **视觉辅助**: 图表、动画、高亮等视觉辅助的效果？
- **速度控制**: 关键步骤是否放慢？简单步骤是否加速？

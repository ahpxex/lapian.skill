# 悬疑/推理 (Mystery)

叠加在 `_base.md` 之上的题材特化分析维度。

## 1. 信息揭露设计

- **信息层级**: 故事包含几层真相？每层何时揭示？
- **线索布局**: 关键线索是公平呈现还是刻意隐藏？事后回看是否合理？
- **红鲱鱼**: 误导线索的设计质量——是巧妙的误导还是廉价的欺骗？
- **揭示时机**: 真相揭露在结构中的位置？是否给观众足够的消化时间？

## 2. 叙述可靠性

- **叙述者可信度**: 谁在讲述？叙述者是否可靠？不可靠叙述的暗示在哪？
- **视角限制**: 观众的信息被限制在谁的视角？视角切换的规则？
- **记忆与感知**: 是否利用记忆偏差、感知错误作为叙事工具？

## 3. 推理结构

- **推理链**: 从线索到结论的逻辑链是否成立？有没有跳跃？
- **公平性**: 观众是否有机会在揭示前自行推理出真相？（诺克斯十诫 / 范达因规则的遵守程度）
- **反转质量**: 反转是逻辑驱动还是为了反转而反转？反转后故事是否仍然成立？

## 4. 悬念维持

- **问题嵌套**: 主谜题下有多少子谜题？解决一个问题是否打开新问题？
- **时间压力**: 是否有 deadline 机制驱动紧迫感？
- **信息喂养节奏**: 新信息的投放频率——太快则不给消化时间，太慢则失去兴趣？

## 5. 侦探/调查者设计

- **调查方法**: 逻辑推演 / 直觉 / 科学取证 / 社交工程？方法是否与角色一致？
- **调查者的盲区**: 调查者的个人缺陷如何影响调查方向？
- **真相的代价**: 获得真相需要付出什么？认知上的还是情感上的？
